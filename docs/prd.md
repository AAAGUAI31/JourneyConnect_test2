

#  JourneyConnect - 微信小程序开发需求文档（PRD）

## 一、项目概述

**项目名称**：JourneyConnect
**项目定位**：一个帮助旅行者寻找志同道合旅伴（“搭子”）的社交型小程序。用户可以发布或加入旅行活动，查看活动详情，匹配旅行风格相近的人。
**目标人群**：喜欢结伴出游的年轻人、留学生、背包客、短途旅行群体等。
**目标平台**：微信小程序（支持 iOS、Android）

---

## 二、核心功能模块

### 1.  首页（活动广场）

**功能描述**：展示所有已发布的旅行活动卡片流，用户可浏览、搜索、筛选、进入详情。

**主要功能点：**

* 活动卡片展示：

  * 活动标题（如“周末黄山露营”）
  * 活动时间与地点
  * 发起人昵称与头像
  * 已报名人数 / 限制人数
  * 风格标签（如“徒步”“摄影”“轻松游”）
* 筛选与排序：

  * 按时间、地点、旅行风格筛选
  * 支持模糊搜索（关键字匹配标题或地点）
* 悬浮操作按钮：

  * 右下角「➕」按钮 —— 进入“发起活动”页
* 下拉刷新、上滑加载更多

---

### 2.  活动详情页

**功能描述**：展示单个活动的详细信息，用户可报名、查看报名者。

**信息展示：**

* 活动标题、封面图
* 时间与地点
* 游玩行程简介（可分天显示）
* 旅行风格标签（如探险、放松、文化）
* 发起人信息：

  * 昵称、头像、联系方式（微信号或其他）
* 当前报名人数 / 限额
* 已报名人员列表（昵称 + 头像）
* 活动状态（报名中 / 截止 / 已出行）

**交互功能：**

* 「立即报名」按钮（若未登录则提示微信登录）

  * 弹出表单填写基本个人信息：

    * 姓名/昵称
    * 性别
    * 年龄（可选）
    * 联系方式（微信号/手机号）
    * 简短自我介绍
* 「已报名人数」点击展开显示所有报名者

---

### 3.  发起活动页

**功能描述**：允许用户发起新的活动并发布到首页。

**主要字段：**

* 活动标题
* 时间（开始-结束日期）
* 地点（可输入或选择地图定位）
* 游玩行程描述（可按天分段输入）
* 旅行风格（多选标签，如“露营”“潜水”“美食”“徒步”等）
* 人数上限
* 发起人联系方式（默认取微信号，可修改）
* 活动封面图（上传或选择）
* 提交发布按钮（发布后跳转到活动详情页）

**逻辑：**

* 若用户未登录 → 提示授权微信登录
* 成功发布后 → 提示“发布成功”，并返回详情页
* 用户信息自动记录为发起人身份

---

### 4.  我的主页

**功能描述**：展示用户的个人资料与活动参与情况。

**主要内容：**

* 头像、昵称（来自微信授权）
* 微信号或绑定手机号
* 个性签名（可编辑）
* Tab 分区：

  * 我发起的活动
  * 我参加的活动
* 每个活动卡片可点击进入详情
* 编辑个人信息按钮

**未登录状态：**

* 显示登录按钮「微信一键登录」
* 登录后自动刷新个人信息

---
## 三、用户流程

1. **首次进入** → 浏览活动（无需登录）
2. **点击报名 / 发布活动** → 提示微信授权登录
3. **登录后** → 可发起活动或报名他人活动
4. **发起成功** → 跳转到详情页并展示
5. **我的主页** → 查看已发起和已参加活动

---

##四、前端部分

框架与技术栈：

微信小程序原生开发（WXML + WXSS + JS/TS）

UI 库：TDesign Weixin
🔗 官方文档：https://tdesign.tencent.com/miniprogram/getting-started

状态管理：Pinia 或 MobX-miniprogram

网络请求：封装 wx.request() → 统一 API 调用层

开发要求：

所有交互组件、表单、弹窗、按钮、卡片、标签、头像、进度条、Tabs 均统一使用 TDesign 组件。
---
## 五、权限与安全

* 用户必须通过微信授权登录后才能发布或报名

* 活动数据审核机制（后期可加管理员后台）
